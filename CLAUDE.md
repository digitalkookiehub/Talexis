## Project Overview

**Project Name:** Talexis
**Description:** AI-powered talent intelligence platform with hybrid LLM architecture (local + cloud) for interview training, evaluation, and placement.
**Tech Stack:**
- Backend: FastAPI + Python 3.11+
- Frontend: React + Vite + TypeScript
- Database: PostgreSQL + SQLAlchemy
- Auth: JWT (Email/Password)
- UI: Tailwind CSS + shadcn/ui
- AI: Ollama (local LLM) + OpenAI GPT-4o (cloud LLM) + Whisper (STT)

def get_student(db: Session, student_id: int) -> StudentProfile:

## Forbidden Patterns

### Backend
- `print()` → use `logging` module
- Plain text passwords → use bcrypt
- Hardcoded secrets → use environment variables
- `SELECT *` → specify columns explicitly
- Skip input validation → validate all inputs with Pydantic
- Raw SQL strings → use SQLAlchemy ORM

### Frontend
- `any` type → use proper TypeScript interfaces
- `console.log` in production → use proper error boundaries
- Inline styles → use Tailwind CSS classes
- Unhandled async errors → always use try/catch with proper error states
- Direct API calls in components → use service layer + hooks

## Module-Specific Rules

### Authentication Module
- All passwords must be hashed with bcrypt
- Access tokens expire in 30 minutes, refresh tokens in 7 days
- Role-based middleware must protect all endpoints
- Four roles: student, college_admin, company, admin

### Interview Module
- Questions must be generated by local LLM (Ollama), never hardcoded
- Each interview type has configurable max attempts
- Questions must be randomized to prevent cheating
- Audio answers are post-MVP; text-only for MVP

### Evaluation Module
- All evaluation must go through cloud LLM (OpenAI GPT-4o)
- Scores are 0–10 per dimension: communication, technical, confidence, structure
- Never expose raw evaluation prompts or responses to users
- Store all evaluation metadata for audit trail

### Talent Profile Module
- NEVER expose raw interview data to companies
- All candidate profiles must be anonymized (candidate_code, not name)
- Student consent is REQUIRED before profile becomes visible
- Companies see scores and recommendations only

## API Conventions

- All endpoints prefixed with `/api/v1/`
- Use plural nouns for resources: `/interviews`, `/students`, `/talents`
- Return appropriate HTTP status codes:
  - 200: Success
  - 201: Created
  - 400: Bad Request
  - 401: Unauthorized
  - 403: Forbidden (wrong role)
  - 404: Not Found
  - 409: Conflict
  - 429: Rate Limited

---

## Authentication

### JWT Configuration
- Access token expires: 30 minutes
- Refresh token expires: 7 days
- Algorithm: HS256
- Tokens stored in httpOnly cookies (frontend)

### Role-Based Access Control
| Role | Access |
|------|--------|
| student | Own profile, interviews, readiness, learning |
| college_admin | All students in college, analytics |
| company | Talent profiles (anonymized), jobs, shortlists |
| admin | Everything + user management + system settings |

---

## AI/LLM Configuration

### Local LLM (Ollama)
- Model: llama3 or mistral (configurable)
- Used for: question generation, resume parsing, preprocessing
- Endpoint: http://localhost:11434

### Cloud LLM (OpenAI)
- Model: gpt-4o (configurable)
- Used for: evaluation, scoring, feedback, recommendations
- Must handle rate limits and retries gracefully

### Prompt Management
- All prompts stored in `backend/app/services/llm/prompts.py`
- Prompts must be versioned (include version in metadata)
- Never hardcode prompts inline in service code
